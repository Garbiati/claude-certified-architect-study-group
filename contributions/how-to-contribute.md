# Como Contribuir

Obrigado pelo interesse em contribuir com este repositório! 🙌

Este projeto é mantido pela comunidade e toda contribuição é bem-vinda — desde a correção de um erro de digitação até a adição de novos materiais de estudo completos.

---

## 🛠️ Tipos de Contribuição

- **📝 Conteúdo de estudo**: Notas, resumos, explicações de tópicos do exame
- **🔗 Recursos**: Links para artigos, vídeos, cursos ou documentação relevante
- **🌍 Traduções**: Tradução de materiais para pt-BR, inglês ou espanhol
- **🐛 Correções**: Erros de digitação, links quebrados, informações desatualizadas
- **💡 Melhorias**: Sugestões de estrutura, formatação ou novos tópicos
- **📋 Notas de sessão**: Resumos e aprendizados das sessões de estudo

---

## 📋 Passo a Passo para Contribuir

### 1. Faça um Fork do Repositório

```bash
# Clique em "Fork" no GitHub e clone seu fork localmente
git clone https://github.com/SEU-USUARIO/claude-certified-architect-study-group.git
cd claude-certified-architect-study-group
```

### 2. Crie uma Branch

Use um nome descritivo para sua branch:

```bash
# Para adição de conteúdo
git checkout -b content/adicionar-resumo-rag

# Para correções
git checkout -b fix/corrigir-link-recursos

# Para traduções
git checkout -b translation/en-study-plan
```

### 3. Faça suas Alterações

- Escreva em **Português (pt-BR)** sempre que possível
- Siga a estrutura e formatação Markdown dos arquivos existentes
- Adicione seu conteúdo na pasta/arquivo mais adequado

### 4. Confirme suas Alterações

```bash
git add .
git commit -m "feat: adicionar resumo sobre RAG na semana 3"
git push origin content/adicionar-resumo-rag
```

### 5. Abra um Pull Request

- Acesse o repositório original no GitHub
- Clique em "New Pull Request"
- Selecione sua branch
- Preencha o título e a descrição com clareza
- Aguarde a revisão da comunidade

---

## ✍️ Guia de Estilo

### Idioma

- **Português (pt-BR)** é o idioma principal
- Contribuições em **inglês** são aceitas
- Use o mesmo idioma em todo o documento (não misture idiomas no mesmo arquivo)

### Markdown

- Use títulos hierárquicos (`#`, `##`, `###`)
- Use listas para enumerações
- Use tabelas para comparações
- Use blocos de código para exemplos técnicos
- Use negrito e itálico com moderação

### Conteúdo

- Seja claro e objetivo
- Cite as fontes quando referenciar materiais externos
- Não inclua conteúdo proprietário ou com direitos autorais
- Prefira fontes públicas e verificáveis

## 📝 Convenções de Commit

Use o padrão [Conventional Commits](https://www.conventionalcommits.org/):

| Prefixo | Uso |
|---------|-----|
| `feat:` | Novo conteúdo ou funcionalidade |
| `fix:` | Correção de erros |
| `docs:` | Atualização de documentação |
| `translation:` | Tradução de conteúdo |
| `chore:` | Tarefas de manutenção |

**Exemplos:**
```
feat: adicionar tópico sobre HyDE no guia de RAG
fix: corrigir link quebrado em resources.md
translation: traduzir study-plan para inglês
docs: atualizar cronograma de sessões
```
